装环境, 上线流程

装环境

一共需要装node, git, svn, nginx

工具的话用sublime和chrome, 还可以装个git和客户端提交时方便, charles用来代理连手机

可以参照我们前端组的wiki: http://wiki.corp.ganji.com/%E5%89%8D%E7%AB%AF%E7%BB%84/%E5%85%B3%E4%BA%8ENG-static

ubuntu里面用npm来装, mac里用brew来装

brew在终端用命令 ruby -e “$(curl -fsSL https://raw.github.com/mxcl/homebrew/go)”, 就可以装上, 之后想安装点什么东西就用 brew install xxxx 就可以

brew install node

brew install nginx

brew install svn

brew install git

ubuntu里面用npm就是 npm install xxxx 都一样的


这些工具都装上之后就可以把公司项目的代码拉到本地来, 可以先建个workplace文件夹, 以后工作的代码就都在这个文件夹里面, 进这个文件夹, 用命令 git clone 把(地址在wiki里面有), 拷到自己的电脑上
之后git里面所有的子模块都要先初始化, git submodule update –init , 再之后把所有的子模块都切换到 online 上, 不然之后会有乱问题, 可以用命令 git submodule foreach ‘git checkout online’ 但是这个命令可能执行不了, 那就手动一个个切 online , 基本的工具包括 ng_sta/src/app/che 下的client, ng_sta/src/com 下的g, mobile, 里面的lib,然后这些目录就一直保持在online 分支上干各种工作就对了(等牛逼那一天就有可能轮到改这里的文件了)

然后编译这里的文件, 用 grunt build, 全通过就对, 编译时候可能有不通过的地方也没关系, 进入到有问题的那个目录, checkout到online分支上pull一下, 然后再编译就能过了, 生成了一个build目录.

再之后要改nginx的配置, 一共两个地方, 输入 nginx -h 可以得到nginx安装的目录, 进这个目录改nginx.conf这个配置文件, 直接换一份就行; 还有就是在nginx目录下创建一个sites文件夹, 在里面 ganji_sta.conf这个配置文件, 要把里面的路径都改成自己的工作路径,
然后环境就好了, 进到ng_sta里用grunt watch, 再保存ng_sta里的文件里时候就会自动编译, 生成编译后的文件在build目录里了.

上线流程

可以先建一个jira, 在apollo里右上角有个”创建问题”, 选择”web前端”, “新功能”, 会得到一个jira号, 再到gitlab里创建分支, “new branch”, 然后”name”就写jira号”feature/WF-xxxx”, “create from”, online, 就建完分支了.


代码写完后, 先把console.log那些都去掉, git commit push完, 在gitlab里能看到操作记录, 有merge request, 选好上到develop, 或者online, 提交merge request, 有权限的同学来merge, 就行了.
