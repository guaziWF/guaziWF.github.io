页面工作流程

小总管

这篇文章给自己读，也给未来的组里的小白们，话说以后会有小白吗？不会就我们两只吧？满满的都是泪

‘xxxxxxxx/ng/app/che/common/test/echarts_test/index.html#app/che/common/test/echarts_test/quering.js’

以这个链接为例，这个链接是个app页面，在外面加了一层壳

'xxxxxxxxx/ng/app/che/common/test/echarts_test/index.html'

所以访问页面时首先进入这个index.html页面，下面来看html的源码，主要看javascript代码：

1 最先遇到的script标签中，显示了引入的js文件 为g.js和echarts-all，js

2 接下来遇到的script标签中，js代码是判断你所使用的手机是安卓系统还是ios系统

3 最后一部分的script标签中，直接看下面这部分代码 

function onVersionJSReady () {
        G.use(['app/client/common/init.js', 'app/client/common/style/global.jcss'])
            .fail(onFail);
    }

这里显示了g.js依赖于init.js （global.jcss是加载的所有样式，这里不管）。主要看init.js。

到这里是不是已经懂了一半了，来来来，继续。

init.js是页面加载时初始化页面的，这里通过对href的处理，获取到链接的hash值等等，具体看init.js的源码中各个变量的值，这里主要讲的是异步加载这个函数。 

require.async([viewPath], function (view) {
    ...
    view.init(config);
    ...
}

查了一些资料得出：它的作用是异步加载模块，并在加载完模块后执行回调函数。这里加载了根据hash值查到的此路径下得文件，然后执行回调函数。view就是加载的文件返回的对象。对view.init(config)进行调试后发现进入的就是quering.js里的init方法，这时就开始加载页面和相关数据。

so 其实流程相当简单，首先进入index.html文件中，看g.js依赖了init.js,进入init.js搞懂每个变量的意义，并发现了异步加载函数通过加载viewPath这个变量路径下得文件得到返回的对象，调用对象提供的init方法……

就是这样……哪里说得不对求指正啊。
