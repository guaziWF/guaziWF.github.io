flexbox 总结

by 小总管

—写在前面的话。

　　我挺喜欢做笔记的，但是很懒，不喜欢整理笔记，这样其实很不好，不能做到温故知新嘛。写这个博客最大的好处就是加深记忆！！

　　大股东说让写flexbox的笔记，当时真想说没听到没听到，主要是感觉对flexbox的感悟不多，感悟最多的是大股东吧，毕竟他都用flexbox写了个react native的例子出来。好吧说了这么多还是要写的，下面就看小白的总结吧。

1.什么是flexbox？ 

我对flexbox的认识就是它是一个弹性布局，它最大的用处就是自适应屏幕，对pc和wap都很适用。发展到现在对各种浏览器都支持。  
flexbox有老版本，新旧混杂的版本，新版本。这里就只介绍一下新版本。  
flexbox是一个新的盒装模型，任何一个容器都可以使用flexbox进行布局。不论是块状元素（block）还是行内元素（inline）。

2.flexbox元素标签。

1.display是否将元素设置为伸缩容器的方法。

display:flex|inline-flex;flex是将元素设置为块伸缩容器，inline-flex是将元素设置为内联伸缩容器。

2.flex-direction伸缩流方向，主要用来创建主轴，从而定义伸缩项目放置在伸缩容器的方向，因此它是伸缩项目的父元素。

flex-direction：row| row-reverse |column |column-reverse;  
row: 在ltr（left-to-right）排版方式下从左向右，在rtl排版方式下从右向左；row-reverse与之相反；

3.flex-wrap伸缩换行，用来创建侧轴，主要是伸缩容器里的伸缩项目是单行还是多行，侧轴的方向决定了新行堆放的方向，它也是伸缩项目的父元素。

nowrap:伸缩项目单行显示；在ltr（left-to-right）排版方式下从左向右，在rtl排版方式下从右向左
wrap:伸缩项目多行显示；在ltr（left-to-right）排版方式下从左向右，在rtl排版方式下从右向左
wrap-reverse:伸缩项目多行显示，但是与wrap排序相反

4.flex-flow 这个是属性2和3的缩写版，同时定义了伸缩容器的主轴和侧轴。它的默认值是row nowrap

flex-flow: <'flex-direction'> || <'flex-wrap'>

5.justify-content 它也是伸缩项目的父元素。主要定义伸缩项目沿着主轴线的对齐方式。当一行的所有哦伸缩项目都不能伸缩或可伸缩但已经达到最大长度时，这一属性才会对伸缩容器额外空间进行分配。当伸缩项目溢出某一行的时候，这一属性也会在项目的对齐上加一些控制

justify-content:flex-start | flex-end |center | space-between | space-around
flex-start:伸缩项目向一行的起始位置靠齐。
flex-end:伸缩项目向一行的结束位置靠齐。
center: 伸缩项目向一行的中间位置靠齐。
space-between: 伸缩项目会平均的分布在行里，其中第一个伸缩项目在一行中最开始的位置，最后一个项目在一行中终点的位置。
space-around:伸缩项目会平均的分布在行里，两端保留一半的空间。

6.align-items和align-self 这是侧轴对齐的两种方式。

align-items: flex-start | flex-end | center | baseline | stretch;主要是伸缩项目行的对齐
baseline: 项目的第一行文字的基线对齐。
stretch（默认值）：如果项目未设置高度或设为auto，将占满整个容器的高度。
align-self：是指伸缩项目自身的对齐

7.align-content 当容器还有额外的空间的时候，align-content可以用来调准伸缩行在伸缩容器的对齐方式。

align-content: flex-start | flex-end | center | space-between | space-around | stretch;
各个属性含义参照6

总结：1-7这几个属性主要是伸缩容器的属性。

下面主要介绍的是伸缩项目自己的属性

1.flex

flex: none | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'> ]
flex-grow:定义项目的放大比例
flex-shrink: 定义项目收缩的能力
flex-basis:定义伸缩项目的伸缩基准值，伸缩容器的额外空间按比率进行伸缩。

2.order 显示顺序

可以根据偶然的人属性来改变伸缩项目出现的次序
order：<number>

这些主要是看《图解css3》上的内容，觉得把它和阮大爷的解释一起看比较好，大爷有的讲的貌似有点问题。

页面工作流程

by 小总管

这篇文章给自己读，也给未来的组里的小白们，话说以后会有小白吗？不会就我们两只吧？满满的都是泪

‘xxxxxxxx/ng/app/che/common/test/echarts_test/index.html#app/che/common/test/echarts_test/quering.js’

以这个链接为例，这个链接是个app页面，在外面加了一层壳

'xxxxxxxxx/ng/app/che/common/test/echarts_test/index.html'

所以访问页面时首先进入这个index.html页面，下面来看html的源码，主要看javascript代码：

1 最先遇到的script标签中，显示了引入的js文件 为g.js和echarts-all，js

2 接下来遇到的script标签中，js代码是判断你所使用的手机是安卓系统还是ios系统

3 最后一部分的script标签中，直接看下面这部分代码 

function onVersionJSReady () {
        G.use(['app/client/common/init.js', 'app/client/common/style/global.jcss'])
            .fail(onFail);
    }

这里显示了g.js依赖于init.js （global.jcss是加载的所有样式，这里不管）。主要看init.js。

到这里是不是已经懂了一半了，来来来，继续。

init.js是页面加载时初始化页面的，这里通过对href的处理，获取到链接的hash值等等，具体看init.js的源码中各个变量的值，这里主要讲的是异步加载这个函数。 

require.async([viewPath], function (view) {
    ...
    view.init(config);
    ...
}

查了一些资料得出：它的作用是异步加载模块，并在加载完模块后执行回调函数。这里加载了根据hash值查到的此路径下得文件，然后执行回调函数。view就是加载的文件返回的对象。对view.init(config)进行调试后发现进入的就是quering.js里的init方法，这时就开始加载页面和相关数据。

so 其实流程相当简单，首先进入index.html文件中，看g.js依赖了init.js,进入init.js搞懂每个变量的意义，并发现了异步加载函数通过加载viewPath这个变量路径下得文件得到返回的对象，调用对象提供的init方法……

就是这样……哪里说得不对求指正啊。
